 They `lono new` command sets up the project structure so you know where things go. For a brand new project, most of the folders are still empty.

```sh
$ tree autoscaling
autoscaling
├── app
│   ├── definitions
│   │   └── base.rb
│   ├── helpers
│   ├── partials
│   ├── scripts
│   ├── templates
│   └── user_data
├── config
│   ├── params
│   ├── settings.yml
│   └── variables
└── output
$
```
